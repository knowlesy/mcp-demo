Regenerate the repomix codebase snapshot so it reflects the current repo state.

Run:

```bash
repomix --config repomix.config.json
```

Then stage the result:

```bash
git add repomix-output.xml
```

The output file (`repomix-output.xml`) is committed to this repo so AI tools always have a
current, single-file snapshot of the entire codebase. It is auto-regenerated by a
PostToolUse hook whenever a `git commit` Bash command is detected — but run this
skill manually after bulk file changes that bypass a commit.

Reference policy for new questions:

- For repo-specific, codebase-specific, or workflow-specific questions, consult
	`repomix-output.xml` first.
- For general conceptual questions that do not depend on this repository, do not
	require reading `repomix-output.xml`.
- Re-consult `repomix-output.xml` after substantial repo changes or when answer
	confidence is low due to missing project context.
